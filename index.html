New installer flow

Download the app, not a loose folder.

The primary download is now a macOS DMG. The app carries the extension payload inside the bundle, copies it into Application Support on first launch, and installs the native bridge for local terminal handoff.

  • 1Drag Ask your GIT Companion.app into Applications.
  • 2Open it once to prepare the Chrome extension folder and native bridge.
  • 3For this prototype, load that prepared folder in chrome://extensions.

Desktop + browser

One product, two connected surfaces.

The app installs like a lightweight Mac utility, while the browser extension appears directly on repo pages. Together they turn the current repository into a chat, summary, and local setup workflow.

1

Menu-bar companion

A small macOS app stays in the top bar, installs the browser bridge, and detects current repository pages.

2

Repo page assistant

The browser extension adds Ask your GIT to GitHub, GitLab, and Bitbucket repository pages.

3

Chat before setup

Ask what the repo does, which stack it uses, and how to run it before touching your terminal.

4

Local tool handoff

Send install commands to Claude Code, Cursor, Codex, Terminal, or your own custom tools.

5

Works without a key

The prototype includes local fallback answers when hosted AI is unavailable or no API key is configured.

6

DMG download

The macOS app, icon, native bridge, and extension payload ship together in a single downloadable DMG.

Setup

Five steps to the prototype.

1

Download

Get the Ask your GIT macOS DMG from this site.

2

Install app

Drag Ask your GIT Companion.app to Applications, then open it once.

3

Launch once

The app prepares the extension folder and installs the local native bridge.

4

Load extension

Open chrome://extensions, enable Developer mode, and load ~/Library/Application Support/Ask your GIT/extension.

5

Ask a repo

Open any repo and use Ask AI to understand it before installing.
